✅ AI会議室 MVP 完成ライン（PASS/FAILで判定）
0) 前提（UI凍結）

 UI凍結：ボタン配置・文言・色・サイズを変えていない（内部だけ直す）

判定：見た目スクショ差分が出ない

1) 画像の受け渡し（最重要）

 画像が次のAIで見える（途中で欠けない／0枚にならない）

判定：①貼付→②転送→③受信側で「画像あり」確認

 複数画像でも崩れない（2枚以上）

判定：2枚で同じ手順を通す

 画像が大きくても落ちない（スマホ撮影1枚でOK）

判定：転送時にエラー表示が出ない

2) AI選択（“全員＝空セット”ルール）

 全員選択の挙動が統一（空セット＝全員）

判定：全員ボタン→選択表示→送信先が全員になる

 個別選択の挙動が正しい（選んだAIだけへ）

判定：2人だけ選ぶ→2人だけに発言が出る

 時刻チップ表示が正しい（全員時＝全AI / 個別時＝選択AIのみ）

判定：表示がブレない

3) 転送〜回答が返る（体感の心臓）

 転送したら必ず回答が返る（無言にならない）

判定：3回連続で成功

 “編集→送信”の2段階が崩れない（送信で必ずcallが走る）

判定：送信直後にステータスが動く（待ち状態→結果）

4) ログ保存（後で追える）

 会話ログが残る（更新しても消えない）

判定：リロード後に履歴が残る

 最大件数の上限が効く（各AI 50件など）

判定：増やしても表示が重くならない／古いのが整理される

 画像ログも追える（最低：存在が分かる）

判定：ログに「画像あり」が残る（URL or 件数でもOK）

5) “重くならない”運用（セッション分割）

 長く使っても重くならない（体感でOK）

判定：30分触って入力遅延が増えない

 自動分割しても迷子にならない（フォルダ/スレッド設計）

判定：分割後も「どこが続きか」一目で分かる

6) エラー表示（ブラウザダイアログ禁止）

 alert / confirm / prompt を使わない（見た目が安っぽくならない）

判定：代わりに画面内メッセージ or トーストで出る

 失敗理由が短く出る（初心者でも次の手が分かる）

判定：「何をすれば良いか」が1行で分かる

🧪 テスト手順（なべちょうさん式：A→C→B）

Test A（画像）→ Test C（ログ）→ Test B（AI選択） の順でやると早いよ。

Test A：画像1枚→転送→受信側で画像確認（3回）

Test C：リロード→ログ残るか確認（2回）

Test B：全員（空セット）→個別2名→表示と送信先確認（各1回）

🧾 記録テンプレ（1行でOK）

結果：A=PASS/FAIL、C=PASS/FAIL、B=PASS/FAIL

メモ：失敗した場所（例：画像2枚で0枚になる、全員選択が崩れる、など）
