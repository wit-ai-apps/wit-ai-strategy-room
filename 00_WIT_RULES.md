WIT PROJECT RULES（UI固定・納品ルール）
0) 最優先：UIは絶対に変えない

画面の見た目（配置/余白/色/文字/ボタン/サイズ）を変更しない

HTML構造（要素の追加・削除・入れ替え）をしない

class名 / id名 / 表示文言 を変更しない

UI変更が必要な場合は、作業を止めて「UI変更が必要」と報告する（勝手にやらない）

1) 変更して良いのは“中身だけ”

JavaScriptの内部処理、バグ修正、通信処理、ログ、エラー対策のみOK

CSSの見た目調整は禁止（UIを変えないため）

2) 納品形式

断片ではなく「完全な1ファイル」を出す

先頭にヘッダーを入れる（VERSION / BUILD / DATE(JST) / AUTHOR / CHANGES / BUILD_PARAM / DEBUG_PARAM）

3) 検証（最低ライン）

5回以上の再現チェック（同じ操作を繰り返して崩れないこと）

UIが変わっていないことを最優先で確認する

4) 報告フォーマット（必須）

VERSION / BUILD_ID / AUTHOR

変更点（UI非変更を明記）

テスト項目（何を何回やったか）

残リスク（あれば）
